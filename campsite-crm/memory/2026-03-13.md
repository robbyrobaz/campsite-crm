### Tablet Camera Sizing Battle (45+ minutes, 20+ iterations)
**Problem:** Bottom camera cut off on tablet, had to scroll

**Root causes discovered:**
1. Browser caching was extremely aggressive — CSS changes not applying
2. Viewport height calc needed to account for: header (43px), section title, camera info sections, gaps
3. Rob's tablet screen size required precise tuning

**Attempts (all initially defeated by caching):**
- `max-width: 94%` → too small
- `max-width: 85%` → still too small
- `transform: scale(0.9)` → just added white space
- `gap: 4px` → didn't fix height issue
- Multiple `calc((100vh - Xpx) / Y)` iterations

**Final solution (that actually worked after cache fix):**
- CSS: `height: calc((100vh - 280px) / 3)`
- Added cache-busting: timestamp injected into CSS on every page load
- Cache headers: `Cache-Control: no-cache, no-store, must-revalidate, max-age=0`
- Imported `make_response` from Flask (was missing, caused 500 errors)

**Cache-busting code:**
```python
html = DASHBOARD_HTML.replace('<style>', f'<style>/* v{int(time.time())} */')
```

### Key Lessons
1. **Browser caching is BRUTAL** — even with no-cache headers, CSS wasn't updating. Timestamp injection was required.
2. **Viewport calculations** — must account for ALL UI chrome: `(100vh - [all headers/padding]) / [row count]`
3. **Never assume CSS changes applied** — always verify by loading page in browser and inspecting
4. **Port conflicts** — 8558 (ring-mqtt go2rtc), 5000 (node.js), 8888 (MediaMTX HLS) all had conflicts
5. **IP hardcoding breaks everything** — use `window.location.hostname` for portability
