## [10:32] OpenClaw Memory System Deep Dive

Rob asked what's automatic vs manual in OpenClaw's memory system. Read the actual docs (`docs/concepts/memory.md`, `docs/reference/memory-config.md`).

**Key findings:**
- **Auto-injected:** MEMORY.md + memory/*.md into system prompt (Project Context)
- **Auto:** memory_search/memory_get tools, file watching + reindexing, pre-compaction memory flush
- **NOT auto:** Writing to memory files — agent must do this manually
- **Pre-compaction flush:** When session nears context limit, OpenClaw triggers silent agentic turn to write memories. Enabled by default. Does NOT fire on `/new` or idle reset — only compaction.
- **Session memory indexing:** `memorySearch.experimental.sessionMemory: true` can index old transcripts as searchable memory. Not enabled yet.
- **Config:** Rob's openclaw.json has zero compaction or memorySearch overrides — all defaults apply (which is fine for most things).
