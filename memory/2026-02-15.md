## New System: Strategy Evolution v2 (Proposed)

### Core Concept
- Maintain 20 active test strategies at all times
- 48-hour testing windows per strategy
- Each strategy gets up to 3 tuning attempts (Sonnet)
- If 3 adjustments fail to improve, archive it
- Opus auto-designs replacement based on failure patterns + market regime

### Lifecycle Loop
```
Every 48 hours:
1. Score all 20 strategies (Haiku)
2. Find stalled strategies (no improvement for 2+ rounds)
3. For stalled: tune (Sonnet) or archive + replace (Opus designs)
4. Log changes, failure reasons, design prompts
5. Report: "Promoted X, tuned Y, replaced Z"
```

### Key Design Points
- **Pool size:** 20 (diversity without overload)
- **Adjustment limit:** 3 per strategy
- **Improvement threshold:** +5% score increase
- **AI allocation:** Haiku (scoring), Sonnet (tuning), Opus (design only)
- **Full automation:** No human design input, only AI-generated candidates

### Database additions needed
- `strategy_lifecycle`: generation, status, adjustment_round, scores, failure_reason
- `strategy_design_history`: design_prompt, market_context, inspired_by_failure
