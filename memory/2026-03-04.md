## Critical Learning: Backtest vs Training (03:59 MST)

**🚨 MAJOR CLARIFICATION — Rob caught this mistake:**

In the NQ Pipeline, "backtest" and "training" are **two distinct operations**:

### Backtest (run_phase2.py)
- Runs `strategy.generate_signals()` on historical data
- Evaluates signal logic across millions of historical bars
- Generates statistics: PF, WR%, trades, eval_pass_rate, etc.
- **Writes statistics to the registry database**
- **Does NOT generate pkl files for live inference**

### Training (ml/retrain_strategies.py)
- Takes the tested signal logic and trains actual ML models
- Generates pkl files: `ml/models/{strategy}/{strategy}_live_v1.pkl`
- These pkl files are what the watcher actually uses to run strategies live

### What I got wrong (03:00-04:00 MST)
1. ✓ Created 5 strategy files
2. ✓ Ran them through backtest tournament (4.5 hours)
3. ✓ Got backtest statistics proving edge (PF~2.04, eval=100%)
4. ❌ **Immediately wired them into STRATEGIES list WITHOUT pkl files**
5. ✓ Then created training card to generate the pkl files

**The mistake**: Wiring before training left the strategies in "OFF" status because no pkl files existed.

**What the training card fixes**: Generates the pkl files, which enables watcher to load them and transition to "LIVE" status.

### Correct Workflow (Going Forward)
1. Create strategy Python file
2. Run backtest with run_phase2.py → get statistics
3. **Train the strategy** with ml/retrain_strategies.py → get pkl files
4. Wire into STRATEGIES list
5. Restart watcher

Do NOT wire before training.

## ROB'S CLARIFICATION ON WORKFLOW (04:03 MST) — CRITICAL

**You are 100% correct. I was wrong.**

**The actual NQ workflow per PRD:**
- Backtest and training ARE THE SAME OPERATION
- `run_phase2.py` does walk-forward training on historical data:
  - Train 120 days → validate 30 days → test OOS 20 days
  - Model learns during training, evaluated on OOS folds
  - Metrics (PF, WR, eval_pass_rate) come from model evaluation
- Gates check: PF≥1.5, trades≥50, DD<$3K, eval_pass_rate≥40%
- **If gates pass, the trained model should be saved as the pkl file**
- No separate "training" step after backtest

**What I incorrectly did:**
- Treated backtest (run_phase2.py) as "get metrics only"
- Then created a separate "training card" to train models
- This is backwards — run_phase2.py IS the training

**What should happen:**
- `run_phase2.py --strategies X Y Z` runs walk-forward backtest
- During that process, it trains models and evaluates them
- If a strategy passes gates, `run_phase2.py` should automatically:
  - Retrain final model on all historical data
  - Save to `ml/models/{strategy}/{strategy}_live_v1.pkl`
- Wire into STRATEGIES list
- Restart watcher

**The implication:**
Either `run_phase2.py` needs to be modified to save pkl files on gate pass, OR the current implementation already does this and I just didn't see it.
