## Backup System Issues Summary (carried from Mar 29)

Three related problems, all caused by blofin_monitor.db being ~6GB:
1. **GFS rotation rarely runs** — script calls `gfs_rotate()` AFTER all DB backups. blofin_monitor takes 60+ min, next timer preempts before rotation. Self-heals occasionally.
2. **Service timeout** — FIXED (45min → 90min)
3. **Concurrent backups** — hourly + daily both backup blofin_monitor simultaneously. Wasteful.

**Proper fix:** Restructure backup script — run `gfs_rotate()` FIRST, skip blofin_monitor if another backup in progress, or move blofin_monitor to its own less-frequent schedule.
