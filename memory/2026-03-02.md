# 2026-03-02 Session Log

## Major Achievement: Complete NQ Tournament Pipeline

### The Problem We Solved
Research was happening in a vacuum. We'd find insights (trailing stops, session-aware exits) but never properly integrate them into the competition. Results would sit in separate tables, never flowing to forward test.

### The Solution: Research → Backtest → Tournament → Promotion → Forward Test

Built a complete automated pipeline:

1. **Tournament** (`tournament.py`): Evaluates all strategies every 6h
2. **Session-Exit Sweep** (`session_exit_sweep.py`): Tests 54-972 exit configs per expert per session
3. **Challenger Registration**: Winning configs register in `god_model_tournament` table
4. **Auto-Promotion** (`promote_challengers.py`): Configs that beat champion by >10% auto-update `god_model_config.json`
5. **Service Restart**: nq-smb-watcher automatically restarts with new config

### Rob's Staged ATM Logic (The Key Insight)

Rob's live trading was showing dramatically better results than our forward test:
- psych_levels: 17% WR in test → 53% WR live, PF 1.99, +$1,615
- equal_tops_bottoms: 0% WR in test → 73% WR live, +$345

The difference: His ATM trailing stop with staged tightening.

**Rob's Live ATM Specification:**
```
Stage 0 (Entry):   SL = -60 ticks (-15 pts), TP = +180 ticks (+45 pts)
Stage 1 (Tighten): At +40 ticks (+10 pts) MFE → SL moves to -20 ticks (-5 pts)
Stage 2 (Lock):    At +80 ticks (+20 pts) MFE → SL moves to +20 ticks (+5 pts)
Stage 3 (Trail):   30 tick (7.5 pt) trailing from highest favorable
```

This logic:
- Reduces open risk from -15 → -5 pts early
- Locks profit at +5 pts before trailing
- Trails 7.5 pts from high (captures 30-60% of impulse MFE)

### Key Lessons

1. **Entries were fine, exits were broken** - Our ML models find good setups. The exit logic was the problem.

2. **Staged tightening > simple trailing** - Rob's 3-stage approach (tighten → lock → trail) dramatically outperforms single-activation trailing.

3. **Session-aware exits matter** - Overnight/Asia sessions need different params than NY Midday.

4. **The pipeline must be closed-loop** - Research that doesn't flow to forward test is theater.

## Auto-Card Generator Bug Fixed (10:02 AM)

**Bug:** `AUTO_CARD_GENERATOR.md` instructed the cron to call `/api/cards/<id>/run` immediately after creating cards, setting them to "In Progress" with no actual builder. This happened every hour, causing phantom In Progress cards on every heartbeat.

**Fix:**
- Cards now land in **Planned** only (PATCH status=Planned, no /run call)
- Gate changed from "In Progress >= 6" to "In Progress + Planned >= 3"
- Absolute Rules updated: NEVER call /run, NEVER set In Progress
