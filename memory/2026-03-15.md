## Blofin Backfill Optimization — Research Before Assumptions (16:06-16:11 MST)

### Problem
Historical backfill was taking 85+ hours. Rob asked if we were sure about the rate limits.

### What I Did Wrong (STUPID ASSUMPTIONS)
1. ❌ **Made up batch size:** Set `BATCH_LIMIT = 100` without testing → actual max is **1000** (10x faster!)
2. ❌ **Made up rate limit:** Set to 4 req/sec based on "probably 10 req/sec" assumption → actual is network-limited ~4 req/sec (no API limit hit)
3. ❌ **Added unnecessary sleep:** 250ms between requests when network latency already provides natural throttling

### What I Should Have Done
1. ✅ **Test with curl first:** `curl "...?limit=1000"` → would have discovered 1000 works
2. ✅ **Measure actual throughput:** 50 requests with no sleep → 3.8 req/sec (network bound)
3. ✅ **Check existing working code:** Moonshot v2 had similar assumptions, but we should have TESTED

### Fix
- `BATCH_LIMIT`: 100 → **1000**
- `REQUEST_SLEEP_MS`: 250 → **0**
- Result: **10x faster** (~8 hours total instead of 85 hours)

### CRITICAL LESSON: NO ASSUMPTIONS ON EXTERNAL APIS
**Before configuring ANY external service:**
- Check actual API docs (don't guess)
- Test with curl/requests (verify what it supports)
- Look for existing working implementations
- NEVER make up rate limits, batch sizes, or capabilities

**If you don't have a doc reference or test result, you don't know it. Research first, code second.**
