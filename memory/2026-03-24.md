## Key Lessons — AI Music Video Saga

1. **Research the tool's actual capabilities BEFORE installing**
   - EchoMimicV2: face-only animation
   - PersonaLive: video-to-video (wrong input type)
   - MuseTalk: lip-sync only (not full music video)
   - LTX-2-OPTIMIZED: audio+video generation (CORRECT)

2. **Read the README/examples first, install second**
   - PersonaLive took 2.5 hours to discover it was wrong
   - Could have been 5 minutes of reading

3. **Docker for AI is mandatory BUT painful**
   - MuseTalk: dependency hell for 2h → Docker solved it
   - Then: Docker GPU issues for another 2h
   - Balance: Docker solves dependency hell, adds GPU complexity

4. **GPU memory is a hard constraint**
   - RTX 2080 Super (8GB) can't run MuseTalk float16 mode
   - Background processes fragment VRAM
   - CPU fallback works but 10-20x slower

5. **Open-source AI tools lie about requirements**
   - "Just pip install" → 5 hours of dependency hell
   - Pre-built wheels only exist for old PyTorch versions
   - HuggingFace spaces exist because local install is broken

6. **Time vs money tradeoff**
   - VidMuse: $31, works now, 10 minutes
   - Local build: $0, 13+ hours, maybe works eventually
   - Rob chose local (has Jarvis, wants to learn, 1 video/week)

7. **"Face-only" is NOT a music video**
   - Took 12 hours to realize MuseTalk output is wrong format
   - Rob wanted: multi-scene narrative with camera work
   - Tools provided: static face with moving lips

8. **The >1GB deletion rule exists for THIS reason**
   - Deleted working Docker container mid-use (46GB)
   - Cost 40 minutes to rebuild
   - Rob had to re-state the rule explicitly
   - Large files = critical systems, always ask first
