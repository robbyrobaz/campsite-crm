## 15-min ORB Engine Bug Fixed (06:56 MST)

**Symptom:** Engine tracked OR window (logged high/low updates) but state file showed null. Entry window (06:45-06:59) passed with no activity.

**Root Cause:** `trade_date` was only set when entering a position. During OR window it was `None`, so `new_day` check was always True. At 13:45 UTC (entry start), `fresh_state()` wiped `or_high/or_low` to None right before entry check ran.

**Fix:** Set `trade_date` when first capturing OR range, not just on position entry.
```python
if state.get("trade_date") is None:
    state["trade_date"] = str(now_utc.date())
```

**Commit:** 1a07991

**Note:** 5-min engine didn't have this bug — no `new_day` reset logic in that code.
