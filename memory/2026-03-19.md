## Blofin 1-Min Backfill — Jarvis Takeover (19:00-19:49)

### Problem
The 1-min candle backfill has been attempted ~60 times since Mar 15 with 7 different scripts, 43 log files, and never completed past 107/469 symbols. Crypto agent was monitoring but confused moonshot 4h candle DB (934K rows) with the actual backfill parquets and declared it "complete" when it wasn't.

### Root Cause
- Multiple scripts writing to different locations with different schemas
- Threading caused rate limit crashes
- No resume logic — every restart started from scratch
- Crypto agent watchdog kept restarting the same broken script

### What Was Done
1. **Archived** 12 old scripts + 39 log files → `blofin-stack/archive/backfill_legacy/`
2. **Built new clean system** → `blofin-stack/backfill/backfill_1min.py`
   - Sequential (no threading), 120ms between calls
   - Resume-capable (skips completed symbols)
   - One `candles_1min.parquet` per symbol (not daily files)
   - Storage: `/mnt/data/blofin_tickers/raw/{SYMBOL}/tickers/candles_1min.parquet`
3. **Migrated existing data** — 107 symbols from daily parquets → single files
   - 50.6M unique candles salvaged, ~200M dupes removed
   - 5.5GB → 808MB after dedup + zstd compression
4. **Started backfill** — PID 1312047, ~361 remaining symbols, ETA ~8-10 hours
5. **Hourly Opus monitor cron** — job ID `2c728169-a09a-477b-8284-9c63dc59da12`
   - Fires into main session, checks process alive, restarts if dead
   - Auto-disables when complete
   - Full context embedded (no session memory dependency)
6. **Fired crypto agent from backfill** — told to focus on Moonshot + pipeline only
