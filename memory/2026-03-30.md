## [01:27] Backup Service Timeout ÔÇö openclaw-backup.service FAILED

**What happened:** systemd killed `openclaw-backup.service` after 45 min timeout (`TimeoutStartSec=2700`). blofin_monitor.db backup takes 60+ min (DB is ~6GB), exceeds the limit. The sqlite3 process was killed too.

**Impact:** blofin_monitor.db was NOT backed up this cycle. Other DBs (moonshot_v2, moonshot, kanban, nq_pipeline, energy_data) completed fine before the timeout.

**This is the SAME root cause as the GFS rotation issue:** blofin_monitor.db backup duration exceeds the hourly timer interval AND the service timeout.

**Fix needed:** Increase `TimeoutStartSec` to at least 5400 (90 min) or 7200 (2h) in the systemd unit. Or better: restructure the backup script to handle blofin_monitor separately.

**Not fixed yet** ÔÇö flagged for Rob. Late night, non-emergency (other DBs still backed up, GFS has recent blofin copies from earlier successful runs).

## [02:35] Fixed Backup Timeout

Backup service failed TWICE (01:27 and ~02:30) due to `TimeoutStartSec=2700` (45min). Fixed:
- Increased `TimeoutStartSec` from 2700 to 5400 (90min) in `/home/rob/.config/systemd/user/openclaw-backup.service`
- `systemctl --user daemon-reload && systemctl --user reset-failed openclaw-backup.service`
- Failed service count dropped from 7 back to 6 (baseline)

## [03:35] Observed: Dual Concurrent blofin_monitor Backups

Hourly timer (02:42) and daily timer (03:00) both started `sqlite3 .backup` on blofin_monitor.db simultaneously. Both produce valid copies (SQLite .backup is safe concurrent) but doubles I/O and slows both down. Low priority ÔÇö backup script should check for existing backup process before starting another.

## Backup System Issues Summary (carried from Mar 29)

Three related problems, all caused by blofin_monitor.db being ~6GB:
1. **GFS rotation rarely runs** ÔÇö script calls `gfs_rotate()` AFTER all DB backups. blofin_monitor takes 60+ min, next timer preempts before rotation. Self-heals occasionally.
2. **Service timeout** ÔÇö FIXED (45min Ôćĺ 90min)
3. **Concurrent backups** ÔÇö hourly + daily both backup blofin_monitor simultaneously. Wasteful.

**Proper fix:** Restructure backup script ÔÇö run `gfs_rotate()` FIRST, skip blofin_monitor if another backup in progress, or move blofin_monitor to its own less-frequent schedule.
